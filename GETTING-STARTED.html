09 â€” Gotchas

Things to Watch Out For

Nuxt 4 â‰  Nuxt 3

This project uses Nuxt 4. Some Nuxt 3 documentation may not apply â€” particularly around directory structure (source lives in app/, not at the root).

Always use --envName for builds

A bare nuxt build produces no D1 binding. Always use the project scripts (bun run build or bun run build:staging) which pass the correct --envName.

bun build vs bun run build

bun build invokes Bun's own bundler. bun run build runs the build script from package.json, which is what you want.

D1 bindings in two places

D1 database bindings are configured in both the wrangler config files (for CLI commands like migrations) and in nuxt.config.ts (for the Nuxt/Nitro build). If you change one, update the other.

Auto-imports â€” don't manually import components

Nuxt auto-imports components from app/components/ and composables from app/composables/. You don't need manual import statements for these.

Trunk is a dev dependency, not global

Trunk (the linter orchestrator) is installed as a dev dependency. Invoke it via bun run trunk or install it globally by following the Trunk docs.
